# 本地脑安装与语音对话引导

面向真机用户：从 0 到「本地文字 / 语音对话」。  
App 内三处引导会联动：

| 页面 | 引导内容 |
|------|----------|
| **设置 → 本地推理** | 分步引导卡（✅ 进度 + 行内按钮）+「复制诊断信息」 |
| **设置 → 离线语音识别** | ASR 四步清单（路径 / 加载 / 麦克 / 试转写） |
| **设置 → 桌宠 / 语音陪伴** | 就绪清单（Live2D / ASR / TTS / 本地脑）+ 一键下载 |

任一步报错：**本页点「复制诊断信息」**，或复制 Snackbar / 引擎状态 / 路由预览整段反馈。

## 你需要准备什么

| 组件 | 是否必须 | 说明 |
|------|----------|------|
| 完整 MNN 模型文件夹 | 必须 | `config.json` + `llm.mnn` + `llm.mnn.weight` + tokenizer（如 `tokenizer.mtok`） |
| 麦克风权限 | 语音时必须 | ASR 识别 |
| ASR 模型（Sherpa 等） | 语音时必须 | 桌宠页一键下载，或 ASR 页选手动目录 |
| TTS | 语音播报可选 | 桌宠页配置 TTS 目录 + 参考音频 |

> 不要只拷一个 `llm.mnn`。MNNChat 能跑的目录，一般原样整夹导入即可。

推荐体量：

- 手机 8G：优先 **0.5B / 1.5B**
- 手机 12G+：可试更大；**7B Q4** 建议 16G 级

## 步骤 1 · 准备模型包

1. 使用与本 App 相同体系的 **MNN 导出包**（与 MNNChat 兼容的目录结构）。
2. 用文件管理器确认目录内至少有：
   - `config.json`
   - `llm.mnn`（或配置里写的模型名）
   - 对应 weight
   - tokenizer 文件
3. 放到手机任意可读位置。

## 步骤 2 · 本地推理页：导入并开启

路径：**设置 → 本地推理**（或 智能能力 → 本地模型）。

页顶 **分步引导**：

1. **准备模型包** → 点行内「选择文件夹」（或下方 Path 选择）
2. **启用并加载** → 「一键开启本地对话」；按钮变「本地对话已就绪」= 成功
3. **验证文字对话** → 可点「开优先本地」；再去会话发「你好」
4. **语音 ASR** → 「前往 ASR 设置」
5. **语音 TTS** → 「前往桌宠设置」

也可用页面中部手动开关 +「加载模型」。

### 本步常见报错

| 现象 | 可能原因 | 你怎么反馈 |
|------|----------|------------|
| 导入失败 | SAF 权限 / 目录不完整 / 磁盘满 | 点「复制诊断信息」 |
| 已启用但路径为空 | 没选文件夹或导入中断 | 截图路径行 |
| 加载失败 / ERROR | 缺文件、so 不匹配、内存不足 | 诊断信息 +「重试加载」后再截 |
| 一键后仍未就绪 | 包结构不对（只有 mnn 无 config） | 列出模型目录文件名 |

## 步骤 3 · 验证本地文字对话

任选其一：

**A. 会话强制本地**

1. 新建会话，勾选 **本地模型**
2. 发送：`你好`
3. 应走本地生成

**B. 路由优先本地**

1. 引导第 3 步点「开优先本地」，或打开开关 **优先本地路由**
2. 普通会话发：`你好`
3. 看 **当前路由预览** 是否为 LOCAL

## 步骤 4 · 语音：ASR + TTS + 全屏陪伴

### 4.1 ASR（设置 → 离线语音识别）

页顶清单会显示 ✓/序号：

1. 导入 ASR 模型（桌宠页一键下载更省事）
2. 启用并加载
3. 申请麦克风权限
4. 「试转写」或去全屏陪伴说话

### 4.2 TTS（设置 → 桌宠 / 语音陪伴）

1. 看页顶 **就绪清单**：Live2D / ASR / TTS / 本地脑
2. 一键下载 ASR、TTS（可选 Live2D）
3. 自定义路径区可选手动目录 / 参考音频
4. 点「② 前往本地模型设置」补本地脑（若未就绪）

### 4.3 全屏陪伴

对着手机说短句 → ASR 识别 → 本地脑回复 → TTS 读出。

### 本步常见报错

| 现象 | 可能原因 | 反馈内容 |
|------|----------|----------|
| 无识别结果 | 无麦克风权限 / ASR 未就绪 | ASR 页状态卡 |
| 有字无回复 | 本地脑未 READY / 陪伴超时回 stub | 本地推理诊断信息 |
| 有回复无声音 | TTS 未开 / 静音 / speech 被剥空 | 桌宠页 TTS 就绪行 |
| 复读播报 | 旧生成 loop（应已修） | 气泡原文 |

## 步骤 5 · 出问题怎么复制反馈

**优先**：本地推理页 → **复制诊断信息**（引擎 / 路由 / 路径 / 错误）。

或手动带上：

1. 启用开关、引擎状态、模型路径末 2 段、路由预览、最近 Snackbar  
2. 操作路径：例如「一键开启 → 选夹 → 陪伴语音说你好」  
3. 是否 MNNChat 同包可跑  
4. 闪退：crash 页全文，或 App 内日志 `error-*.log` / `crash-*.log`
