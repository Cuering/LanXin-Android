# 本地脑安装与语音对话引导

面向真机用户：从 0 到「本地文字 / 语音对话」。  
任一步报错时，**复制整段 Snackbar / 引擎状态 / 路由预览** 反馈即可。

## 你需要准备什么

| 组件 | 是否必须 | 说明 |
|------|----------|------|
| 完整 MNN 模型文件夹 | 必须 | `config.json` + `llm.mnn` + `llm.mnn.weight` + tokenizer（如 `tokenizer.mtok`） |
| 麦克风权限 | 语音时必须 | ASR 识别 |
| ASR 模型（Sherpa 等） | 语音时必须 | 设置 → 语音 ASR |
| TTS | 语音播报可选 | 设置 → 语音 TTS |

> 不要只拷一个 `llm.mnn`。MNNChat 能跑的目录，一般原样整夹导入即可。

推荐体量：

- 手机 8G：优先 **0.5B / 1.5B**
- 手机 12G+：可试更大；**7B Q4** 建议 16G 级

## 步骤 1 · 拿到模型包

1. 使用与本 App 相同体系的 **MNN 导出包**（与 MNNChat 兼容的目录结构）。
2. 用文件管理器确认目录内至少有：
   - `config.json`
   - `llm.mnn`（或配置里写的模型名）
   - 对应 weight
   - tokenizer 文件
3. 放到手机任意可读位置（下载目录 / 网盘同步目录均可）。

## 步骤 2 · 在兰心里导入并开启

路径：**设置 → 本地推理**（或 智能能力 → 本地模型）。

1. 点 **「一键开启本地对话」**  
   - 若还没模型：会弹出 **选择文件夹**  
   - 选中**整个模型目录**（不是单个文件）
2. 等待 Snackbar：`已导入并开启…` / 按钮变为 **「本地对话已就绪」**
3. 本页应看到：
   - 启用本地推理 = 开
   - 引擎状态 ≈ **READY**
   - 模型路径非空

### 本步常见报错

| 现象 | 可能原因 | 你怎么反馈 |
|------|----------|------------|
| 导入失败：… | SAF 权限 / 目录不完整 / 磁盘满 | 复制 Snackbar 全文 |
| 已启用但路径为空 | 没选文件夹或导入被中断 | 截图本页路径行 |
| 加载失败 / ERROR | 缺文件、so 不匹配、内存不足 | 复制引擎状态 + 点「重试加载」后再截一次 |
| 一键后仍未就绪 | 包结构不对（只有 mnn 无 config） | 列出模型目录文件名发来 |

## 步骤 3 · 验证本地文字对话

任选其一：

**A. 会话强制本地**

1. 新建会话，勾选 **本地模型**
2. 发送：`你好`
3. 应走本地生成（不依赖网络）

**B. 路由优先本地**

1. 本地推理页打开 **优先本地路由**
2. 普通会话发：`你好`
3. 看本页 **当前路由预览** 是否为 LOCAL

### 本步常见报错

| 现象 | 可能原因 | 反馈内容 |
|------|----------|----------|
| 仍走云端 | 未就绪 / 未勾选本地 / 需要 tools 被路由到云 | 路由预览全文 |
| 空白回复 | 清洗过严或生成失败 | 是否出现过「本地离线生成中」 |
| 同一句刷屏 | 旧版 phrase loop（应已修） | 原文截图 + 是否 READY |
| 闪退 | 见下方「闪退」 | logcat / crash 页 |

## 步骤 4 · 语音对话（本地脑 + ASR + TTS）

1. **设置 → 语音 ASR**  
   - 导入 ASR 模型并启用  
   - 授予麦克风权限  
2. **设置 → 语音 TTS**（若要听回复）  
   - 启用系统或本地 TTS  
3. 进入 **全屏陪伴 / 桌宠语音**  
   - 确认本地脑已 READY  
   - 对着手机说一句短话（如「你好」）  
   - 应：识别文字 → 本地脑回复 →（可选）播报  

### 本步常见报错

| 现象 | 可能原因 | 反馈内容 |
|------|----------|----------|
| 无识别结果 | 无麦克风权限 / ASR 未就绪 | ASR 页状态文案 |
| 有字无回复 | 本地脑未 READY / 陪伴超时回 stub | 本地推理页状态 |
| 有回复无声音 | TTS 未开 / 静音 / speech 被剥空 | TTS 页开关状态 |
| 复读播报 | 旧生成 loop（应已修） | 气泡原文 |

## 步骤 5 · 出问题怎么复制反馈

请尽量带上：

1. **本页**：启用开关、引擎状态、模型路径末 2 段、路由预览、最近 Snackbar  
2. **操作路径**：例如「一键开启 → 选夹 → 陪伴语音说你好」  
3. **是否 MNNChat 同包可跑**（你已确认原 MNNChat 无复读，这对定位很重要）  
4. 闪退时：  
   - App 内 crash 页全文，或  
   - `adb logcat -d | grep -E "AndroidRuntime|MnnLanxin|FATAL"` 相关段  

## 与 MNNChat 的差异（开发说明）

| 项 | MNNChat | 兰心（修复后） |
|----|---------|----------------|
| 采样 | 模型 `config.json` 常含完整 sampler | JNI `set_config` 补齐 **penalty + temperature/top_k/top_p** |
| 出口 | 基本原文展示 | `LocalReplySanitizer` 剥 think/元分析 + **折叠复读** |
| 陪伴 | — | 短 maxTokens + 禁止复读 system |

若 MNNChat 正常、兰心仍复读：优先看是否打到含 JNI 采样补丁的构建；并带上气泡原文。
