# 全屏陪伴 · 视觉讲解（真·场景识别）

> **状态：规划 / 未开工**  
> **勿与 #99 混淆。** #99（`docs/scene-sensing.md`）= 默认关 + 确认 Gate + **单次手动拍照** + 本地颜色启发式 → 换背景/mood。  
> **哥哥原意** = 打开摄像头后**边拍边看**，桌宠/兰心根据画面内容识别，并结合用户提问做讲解；落在**全屏陪伴**里。

## 0. 与 #99 关系

| | #99（已合，底座） | 本 epic（目标） |
|--|------------------|----------------|
| 入口 | 设置 → 场景识别 | **全屏陪伴页**显式开关 |

| 识别 | 本地颜色启发式 | **云端多模态**描述/问答；本地仅兜底标签 |
| 落地 | 换背景 / mood | **对话讲解**（帧描述注入当轮上下文） |
| 复用 | Gate、consent、CAMERA 按需、默认关 | 全部复用；不重做权限叙事 |

## 1. 入口与 UI

1. **全屏陪伴**（`CompanionScreen`）顶栏或侧栏：**「看世界」开关**（默认关）
2. 首次开 → 复用 `SceneSensingGate` 隐私确认（或同文案扩展 consent）
3. 开后：画面内嵌 **CameraX 预览**（小窗/画中画，不挡 Live2D 主体对话区）
4. UI 明确状态条：**「正在看」** / 「已暂停」；随时可关，关即停预览+释放相机
5. **禁止**：无确认后台常开；无预览的静默取帧

## 2. 帧策略（推荐先 A，后可选 B）

### 方案 A — 提问时抓帧（P0，省电可控）

```
用户提问 / 语音结束
  → 若「看世界」开且预览就绪
  → 抓 1 帧（降采样 JPEG，边长上限 ~768）
  → 送多模态 Provider 或先本地描述再注入
  → 本轮对话 system/tool 带上画面摘要
  → 模型结合用户问题讲解
```

- 无提问不上传、不分析
- 失败降级：仅文本聊 + 短提示「这轮没看清」

### 方案 B — 定时抽帧 + 变化才更新（P1 体感）

- 间隔例如 3–5s；简单帧差/亮度变化阈值再更新「当前场景摘要」缓存
- 缓存仅内存；用户下一轮提问时带上「最近所见」
- 仍须预览可见；息屏/切走立即停

**默认实现顺序：A → 再评估 B。**

## 3. 多模态调用

| 优先级 | 路径 | 说明 |
|--------|------|------|
| 1 | **云端多模态 Provider** | 已配置且支持 image 的模型（OpenAI 兼容 vision / Gemini 等，跟现有 Provider 对齐） |
| 2 | 本地轻量标签 | 仅颜色/粗分类兜底（可复用 `LocalSceneClassifier`），**禁止假装已有 VLM** |
| 3 | 无可用视觉 | 明确告诉用户「当前模型看不了图」，不编描述 |

接线原则：

- 帧 → 结构化：`{ caption?, labels?, capturedAt }` 注入**当轮** system 或 tool result
- **不**默认写入长期记忆 / 聊天日志附件；可选用户手动「记住这一眼」再另议
- 与 `PetChatResponder` / ChatRouter 同轮；气泡与 TTS 仍走现有听→想→说

## 4. 隐私

- 默认关；Gate 确认文案写清：预览可见、提问时可能上传缩略帧到已配置云端
- 帧**不落盘**；不写 debug 日志原图
- 关闭开关 / 离开全屏陪伴 / 进程后台：**立即** unbind Camera
- 设置里可一键撤销 consent（对齐 #99 `consentGranted`）

## 5. 明确不做

- 无确认后台常开摄像头  
- 发明 Live2D 资源 / 新 exp·motion  
- 把本地启发式包装成「真视觉理解」  
- 与 #99 设置页手动「识别换背景」抢同一产品叙事（可并存：设置页=氛围映射；陪伴页=讲解）
